## Web 界面设计核心规范

基于 Vercel Labs 的 Web Interface Guidelines，以下是核心设计规范：

---

### 1. 可访问性 (Accessibility)
- 优先使用原生 HTML：<button>、<label>、<input> 优于 ARIA
- 标题层级有序：<h2> → <h6> 依次递进，提供跳转到主内容的链接
- 锚点定位：scroll-margin-top 避免标题被固定导航遮挡

### 2. 焦点状态 (Focus States)
- 交互元素必须有可见焦点样式：focus-visible:ring-*
- 禁止：outline: none 无替代方案
- 使用 :focus-visible 而非 :focus（避免点击时显示焦点环）
- 复合控件使用 :focus-within

### 3. 表单 (Forms)
- 设置正确的 autocomplete 和 name 属性
- 使用正确的 type（email、tel、url、number）和 inputmode
- 禁止阻止粘贴（onPaste + preventDefault）
- 标签可点击（htmlFor 或包裹控件）
- 邮箱/验证码字段禁用拼写检查：spellCheck={false}
- 错误信息内联显示，提交时聚焦第一个错误
- 离开未保存内容前警告（beforeunload）

### 4. 动画 (Animation)
- 尊重 prefers-reduced-motion 偏好
- 只动画 transform/opacity（GPU 友好）
- 禁止 transition: all，明确列出属性
- 动画可中断，响应用户输入

### 5. 排版 (Typography)
- 使用 … 而非 ...
- 使用弯引号 " " 而非直引号
- 数字/单位间不换行：10 MB
- 数字列使用 font-variant-numeric: tabular-nums
- 标题使用 text-wrap: balance 防止孤字

### 6. 内容处理 (Content Handling)
- 长文本处理：truncate、line-clamp-*、break-words
- Flex 子元素需要 min-w-0 以允许截断
- 处理空状态——不渲染损坏 UI
- 预期用户输入：短、中、超长三种情况

### 7. 图片 (Images)
- <img> 必须设置明确的 width 和 height（防止 CLS）
- 首屏下方图片：loading="lazy"
- 关键首屏图片：fetchpriority="high"

### 8. 性能 (Performance)
- 大列表 (>50 项) 需虚拟化
- 渲染中禁止布局读取（getBoundingClientRect 等）
- 批量 DOM 读写，避免交错
- 优先非受控输入；受控输入每次按键需低开销
- 预连接 CDN：<link rel="preconnect">

### 9. 导航与状态 (Navigation & State)
- URL 反映状态：筛选、标签页、分页在 query params
- 链接使用 <a> 支持 Cmd/Ctrl+点击
- 有状态 UI 需深链接
- 破坏性操作需确认或撤销

### 10. 触摸与交互 (Touch & Interaction)
- touch-action: manipulation 消除双击延迟
- 模态框/抽屉使用 overscroll-behavior: contain
- 拖拽时禁用文本选择
- autoFocus 谨慎使用，移动端避免

### 11. 安全区与布局 (Safe Areas & Layout)
- 全宽布局需 env(safe-area-inset-*)
- 避免意外滚动条：overflow-x-hidden
- 布局用 Flex/Grid，而非 JS 测量

### 12. 深色模式 (Dark Mode)
- 深色主题设置 color-scheme: dark
- <meta name="theme-color"> 匹配页面背景色
