## React Native 性能优化建议

### 🔴 优先级 1：列表性能（CRITICAL）

| 规则 | 说明 |
|------|------|
| 使用 FlashList | 用 FlashList 替代 FlatList，大列表性能提升显著 |
| Memo 列表项 | 使用 React.memo() 包裹列表项组件，避免不必要的重渲染 |
| 稳定回调引用 | 使用 useCallback 稳定 onPress 等回调函数 |
| 避免内联对象 | 不要在 render 中创建内联 style 对象，使用 StyleSheet.create |
| 提取函数 | 将函数定义移到组件外部或使用 useCallback |
| 优化图片 | 在列表中使用 expo-image，配置合适的尺寸和缓存 |
| 移出耗时操作 | 将昂贵计算移到列表项组件外部 |
| 使用 item types | 异构列表使用 getItemType 提高渲染效率 |

### 🟠 优先级 2：动画（HIGH）

| 规则 | 说明 |
|------|------|
| GPU 属性优先 | 只动画 transform 和 opacity，这些在 GPU 上执行 |
| useDerivedValue | 计算类动画使用 useDerivedValue 而非直接 shared value |
| Gesture.Tap | 手势场景用 Gesture.Tap 替代 Pressable |

### 🟠 优先级 3：导航（HIGH）

| 规则 | 说明 |
|------|------|
| 原生导航器 | 使用 @react-navigation/native-stack 替代 JS 实现的 stack |

### 🟡 优先级 4：UI 模式（HIGH）

| 规则 | 说明 |
|------|------|
| expo-image | 用 expo-image 替代 RN Image，支持缓存和更好的性能 |
| Pressable | 用 Pressable 替代 TouchableOpacity |
| 原生模态框 | 优先使用原生 modal 实现 |
| onLayout 测量 | 用 onLayout 而非 measure() 获取视图尺寸 |

### 🟢 优先级 5：状态管理（MEDIUM）

| 规则 | 说明 |
|------|------|
| 最小化 state | 减少组件订阅的状态范围 |
| Dispatcher 模式 | 使用 dispatcher 模式避免回调重建 |
| 首渲染 fallback | 首次渲染显示 fallback，避免闪烁 |

### 🟢 优先级 6：渲染（MEDIUM）

| 规则 | 说明 |
|------|------|
| Text 组件包裹 | 文本必须放在 <Text> 组件内 |
| 避免 falsy && | 避免 {count && <View/>}，改用三元表达式 |
