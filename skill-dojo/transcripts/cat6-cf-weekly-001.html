## ✅ 任务完成：CodeFlicker 项目周报生成流程说明

### 核心架构
采用 Agent + 脚本 混合编排模式：
- Agent 负责内容创作（读取、生成、确认）
- 脚本 负责技术执行（截图、上传、发送）

### 完整流程（4 步）

Step 1 读取源数据 [Agent] 
    ↓
Step 2 生成内容 [Agent] 
    ↓
Step 3 用户确认 [Agent] ⚠️ 强制不可跳过
    ↓
Step 4 截图发布 [脚本]


| 阶段 | 执行者 | 具体动作 |
|------|--------|----------|
| Phase 1: 读取源数据 | Agent | 从 KIM Docs 读取周报原始内容，解析内部指标、外部 AARRR、各战线进展 |
| Phase 2: 生成内容 | Agent | 更新 HTML 信息图模板（1800px 宽、浅色清爽风格）+ 生成文字版周报 |
| Phase 3: 用户确认 | Agent | 必须先发给用户个人确认，等待"没问题/可以发"后才能继续 |
| Phase 4: 截图发布 | 脚本 | Chrome Headless 截图 → 裁剪白边 → 上传 KIM → 预览 → 发群 |

### 关键脚本命令
# 查看状态（支持断点恢复）
python3 scripts/cf_weekly_orchestrator.py status

# 一键截图+上传+预览
python3 scripts/cf_weekly_orchestrator.py finalize

# 用户确认后发群
python3 scripts/cf_weekly_orchestrator.py send-group


### 核心约束（踩坑经验）
1. 先确认再发群 — 每次必须用户确认后才能发群
2. 字号偏大 — KIM 会缩小 30-40%，指标数字 56px、正文 30px
3. KIM API 字段 — 用 username/groupId，不用 chatType/receiverId
4. Token 端点 — /token/get 而非 /openapi/v2/oauth2/token
