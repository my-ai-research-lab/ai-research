### 🔴 CRITICAL 级别

1. 消除瀑布流请求 (async-)
- 并行请求：用 Promise.all() 处理独立操作，避免串行 await
- 延迟 await：把 await 移到真正需要结果的分支
- Suspense 流式：用 Suspense 边界流式传输内容

2. 包体积优化 (bundle-)
- 避免 barrel imports：直接从源文件导入，不要从 index.ts 导入

---

### 🟠 HIGH 级别

3. 服务端性能 (server-)
- React.cache()：per-request 去重
- LRU Cache：跨请求缓存
- 并行数据获取：重构组件让 fetch 并行化
- 减少序列化：只传必要数据给客户端组件

---

### 🟡 MEDIUM 级别

4. 客户端数据获取 (client-)
- SWR 去重：自动请求去重和缓存

5. 重渲染优化 (rerender-)
- memo 组件：提取昂贵计算到独立 memo 组件
- 函数式 setState：setCount(c => c + 1) 保持回调稳定
- 派生状态在渲染期计算：不要用 useEffect 同步状态
- useTransition：非紧急更新用 transition

6. 渲染性能 (rendering-)
- content-visibility：长列表用 CSS content-visibility
- 提升静态 JSX：把不变的 JSX 提到组件外
- 三元运算符：用 ? : 而非 && 做条件渲染

---

### 🟢 LOW 级别

7. JavaScript 性能 (js-)
- Set/Map O(1) 查找：替代数组 includes
- 合并迭代：多个 filter/map 合成一个循环
- 缓存属性访问：循环内缓存 obj.prop

8. 高级模式 (advanced-)
- useLatest：稳定的回调引用
- 一次性初始化：全局只初始化一次
