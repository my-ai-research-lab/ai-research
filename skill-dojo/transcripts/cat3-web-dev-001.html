## 网页开发工作流标准流程

### 核心流程（四步）

本地验证 → 部署 → 确认生效 → 发送链接


### 详细说明

#### 1. 开发阶段：本地验证优先
- 先在本地生成页面验证，不需要每次都部署
- 多文件项目必须用 HTTP 服务器预览（python3 -m http.server 8080），避免 CORS 错误
- 本地验证通过后再部署

#### 2. 发布阶段：严格按顺序执行
1. 自验证 → 修改完打开页面检查功能正确
2. 部署 → 推送代码到 GitHub（origin + org）
3. 确认生效 → 打开线上页面确认修改已生效
4. 发送链接 → 确认后才把链接发给用户

#### 3. 文件拆分规范

| 预估行数 | 策略 | 文件结构 |
|:---|:---|:---|
| < 150 行 | 允许单文件 | index.html |
| 150-400 行 | 必须拆分 | index.html + styles.css |
| > 400 行 | 强制拆分 | index.html + styles.css + app.js |
| > 800 行 | 分页+拆分 | 多页面 + shared/ 目录 |

硬性约束：单个文件不超过 300 行

#### 4. 强制完成检查（任务结束前必须）

| # | 检查项 |
|:-:|:---|
| 1 | 本次修改是否需要部署到线上？ |
| 2 | 如果需要，是否已执行部署命令？ |
| 3 | 是否已打开线上页面确认修改生效？ |
| 4 | 是否已将线上链接发给用户？ |

#### 5. 技术要点
- 所有网页必须配置 Favicon（与页面主题匹配）
- 生成顺序：HTML（含外链引用）→ CSS → JS → 本地验证 → 部署
- 质量保障：加载本技能时必须同时激活 meta-execution 技能
