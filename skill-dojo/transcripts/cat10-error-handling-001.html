## 错误处理核心模式

基于 error-handling-patterns 技能，错误处理的核心模式包括：

### 1. 错误处理哲学

| 模式 | 特点 | 适用场景 |
|------|------|---------|
| 异常 (Exceptions) | 打断控制流，try-catch | 意外错误、异常情况 |
| Result 类型 | 显式成功/失败，函数式 | 预期错误、校验失败 |
| 错误码 | C 风格，需要自律 | 底层系统编程 |
| Option/Maybe | 处理可空值 | 可能缺失的值 |

### 2. 错误分类

- 可恢复错误：网络超时、文件缺失、用户输入无效、API 限流
- 不可恢复错误：内存溢出、栈溢出、空指针等编程缺陷

### 3. 三大通用模式

1. Circuit Breaker（熔断器）
- 防止分布式系统级联故障
- 三态：关闭(正常) → 打开(拒绝) → 半开(试探恢复)

2. Error Aggregation（错误聚合）
- 收集多个错误而非首错即停
- 适用于表单校验等批量验证场景

3. Graceful Degradation（优雅降级）
- 主功能失败时提供回退
- 如：缓存失败 → 读数据库 → 返回默认值

### 4. 最佳实践

- Fail Fast：尽早校验输入，快速失败
- 保留上下文：包含堆栈、元数据、时间戳
- 有意义的消息：说明发生了什么以及如何修复
- 正确层级处理：在能有意义处理的地方捕获
- 清理资源：使用 try-finally、上下文管理器
- 不要吞掉错误：记录或重新抛出，不要静默忽略
