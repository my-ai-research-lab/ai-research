## AI 幻灯片技能生成流程

### 核心理念

该技能设计的幻灯片面向阅读和分享（而非现场演示），每张幻灯片都能自包含解释，适合社交媒体传播。

### 完整工作流程

输入 → 偏好加载 → 内容分析 → [检查现有内容?] → 确认(1-2轮) → 大纲 → [大纲审核?] → 提示词 → [提示词审核?] → 图片生成 → 合并导出 → 完成


### 9步流程详解

| 步骤 | 名称 | 说明 |
|------|------|------|
| Step 1 | 设置与分析 | 1.1 加载EXTEND.md偏好
1.2 分析内容信号、语言、推荐幻灯片数量
1.3 检查输出目录是否已存在 |
| Step 2 | 确认 ⚠️必需 | Round 1: 风格、受众、幻灯片数、大纲审核、提示词审核
Round 2: 仅当选择"自定义维度"时触发（纹理、色调、字体、密度） |
| Step 3 | 生成大纲 | 基于确认的风格生成outline.md |
| Step 4 | 大纲审核（条件） | 如用户选择审核则展示结构表格 |
| Step 5 | 生成提示词 | 为每张幻灯片生成图像生成提示词 |
| Step 6 | 提示词审核（条件） | 如用户选择审核则展示提示词列表 |
| Step 7 | 生成图片 | 调用AI图像生成，每张10-30秒 |
| Step 8 | 合并导出 | 生成PPTX和PDF文件 |
| Step 9 | 输出摘要 | 展示完成信息和文件列表 |

### 风格系统

支持 16种预设风格，包括：
- blueprint（默认）- 架构/系统设计
- sketch-notes - 教育/教程
- corporate - 商业/投资
- minimal - 高管汇报
- scientific - 科学/医学

以及 4个风格维度 可自由组合：
- Texture: clean, grid, organic, pixel, paper
- Mood: professional, warm, cool, vibrant, dark, neutral
- Typography: geometric, humanist, handwritten, editorial, technical
- Density: minimal, balanced, dense

### 输出目录结构

slide-deck/{topic-slug}/
├── source-{slug}.{ext}      # 源文件
├── outline.md               # 大纲
├── prompts/                 # 提示词目录
│   └── 01-slide-cover.md, ...
├── 01-slide-cover.png, ...  # 图片
├── {topic-slug}.pptx        # PowerPoint
└── {topic-slug}.pdf         # PDF


### 部分工作流选项

| 选项 | 效果 |
|------|------|
| --outline-only | 仅生成大纲 |
| --prompts-only | 生成大纲+提示词 |
| --images-only | 从现有提示词生成图片 |
| --regenerate N | 重新生成指定幻灯片 |
